# AI Polish + Smart Input Design

**Date:** 2026-02-27  
**Status:** Approved  

---

## Goal

Make meal logging feel like talking to a smart assistant, not filling out a form. Two phases executed together.

---

## Phase 1: AI Polish Layer

Every meal gets a clean **title** and **description** generated by AI at parse time — no extra latency, no extra API call.

### What the AI generates (added to existing parse prompt)

- **`title`**: Short, clean label. e.g. `"Scrambled Eggs & Buttered Toast"`
- **`cleanDescription`**: One readable sentence. e.g. `"Two scrambled eggs with buttered whole wheat toast and a glass of orange juice."`

### Data model change

Add `title String?` to `MealLog` in Prisma schema. One migration. Existing meals: no title, fall back to showing description as before.

### Where titles appear

| Surface | Current | After |
|---|---|---|
| Confirmation card | Raw user input | Clean title (large) + cleanDescription below |
| Meal card (detail page) | Raw input truncated | Title as header, description on expand |
| Dashboard meal status tooltip | Raw input | Title |

### Flows affected

- Text input logging (`/api/log/parse`)
- Photo input logging (`/api/log/parse-photo`)
- Family meal logging (`/api/log/parse-family`) — separate title per child

---

## Phase 2: Smart Input Box

Simplify the logging screen to reduce steps and feel more conversational.

### What changes

**Before:** Meal type selector (4 buttons) → text area → OR switch to "Take a photo" tab

**After:** 
- One text area: *"What did [Mason] have for breakfast?"* (placeholder adapts to inferred meal type)
- Photo button inline in the corner of the input
- Meal type **inferred from time of day**, shown as a small tappable pill:
  - Before 10am → Breakfast
  - 10am–2pm → Lunch  
  - 2–5pm → Snack
  - 5pm+ → Dinner
- Tap the pill to change if the inference is wrong
- No separate tabs

### What stays the same

- Confirmation card (same flow, just the card looks better from Phase 1)
- Save flow
- Quick re-log section above the input

## Architecture

### `src/lib/ai/types.ts`
Add to `ParsedMeal`:
```typescript
title: string;
cleanDescription: string;
```

### `src/lib/ai/providers/anthropic.ts`
Add to `parseFoodDescription` prompt — instruct Claude to return two additional fields:
- `title`: 3–5 words, title case, no filler words
- `cleanDescription`: One sentence, third person, complete and readable

### `prisma/schema.prisma`
```prisma
model MealLog {
  // existing fields...
  title       String?   // new — AI-generated clean title
}
```

### `src/app/api/log/parse/route.ts`
Return `title` and `cleanDescription` from Anthropic response.

### `src/app/api/log/parse-photo/route.ts`
Same — add title + cleanDescription to photo parse.

### `src/app/api/log/parse-family/route.ts`
Same — each child's parsed meal gets its own title.

### `src/app/api/log/save/route.ts`
Accept and save `title` field.

### `src/components/log/meal-confirmation.tsx`
Show `title` as large header, `cleanDescription` as subtext. Keep nutrition breakdown below.

### `src/components/dashboard/meal-list.tsx`
Show `title ?? description` as card header. Show `cleanDescription ?? description` as body text.

### `src/app/(app)/log/[childId]/page.tsx`
- Remove meal type selector buttons
- Add time-of-day inference function
- Add small tappable meal type pill
- Move photo button inline with text input
- Update placeholder to be contextual

---

## Success Criteria

1. User types raw input → confirmation card shows clean title + description
2. All new meals in the list show a title, not raw text
3. Logging screen has no meal type selector — type inferred from time, changeable via pill
4. Photo inline — no tab switch needed
5. Build passes, no regressions
